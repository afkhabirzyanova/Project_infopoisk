# Project_infopoisk

# Поисковая система по корпусу порошков

## Описание проекта

Проект представляет собой поисковую систему по корпусу текстов (стишков-порошков).

На вход подается текстовый запрос, на выход — наиболее релевантные документы из корпуса.

Практически поисковая система может быть полезна для ценителей этого жанра. Например, чтобы находить любимые порошки, из которых помнишь только пару слов или только общую идею. Или чтобы найти новые для себя порошки на интересные темы.

---

## Данные

Используется [корпус порошков](https://github.com/tadgeislamins/poroshki_corpus) (1000 текстов), собранный на основе материалов паблика [Порошки](https://vk.com/sandalporoshki).

---

## Предобработка

Для текстов используется:

- приведение к нижнему регистру
- удаление пунктуации
- токенизация

Стоп-слова не удаляются, так как тексты короткие и служебные слова могут быть значимыми.

Дополнительно:

- для BM25 — лемматизация (pymorphy3)
- для Word2Vec — лемматизация + добавление POS-тегов (pymorphy3)
- для fastText — исходные словоформы без лемматизации

Для fastText лемматизация не применялась, потому что fastText уже представляет слово через его символьные n-граммы. К тому же, корпус устроен так, что в нём много словотворчества, редких и нестандартных форм. Если заранее лемматизировать такие слова, можно частично потерять именно ту информацию о форме слова, которую fastText умеет использовать. Собственно, именно из-за особенностей корпуса третьей моделью был выбран именно fastText, который умеет работать со словами, которых нет в его словаре.

## Индексы

### BM25
Классическая модель информационного поиска на основе частот слов.

Реализована с помощью библиотеки `rank_bm25`.

Оценка сходства — классический BM25-score.

---

### Word2Vec
Используется предобученная модель:
- `word2vec-ruscorpora-300` (gensim downloader)

Вектора документов находятся как среднее векторов входящих в них слов.  

Оценка сходства — косинусная близость.

---

### fastText
Используется предобученная модель:

- `cc.ru.300.bin` (Facebook fastText)

Вектора документов находятся как среднее векторов входящих в них слов.  

Оценка сходства — косинусная близость + небольшой бонус за полностью совпадающие слова между запросом и документом.

Бонус за полностью совпадающие слова был добавлен в качестве относительно удачного эксперимента по борьбе с тем, что даже в самом верхе выдачи fasttext выдавал документы, не имеющие ни одного общего слова с запросом. Видимо, это связано с излишним учётом fasttext-ом n-граммных кусков слов и с использованным грубым способом получения векторов документов просто через усреднение входящих в них слов.

## Поиск

Для случаев, когда оценка сходства запроса неположительна ни с одним из документов, то есть когда ничего релевантного не нашлось, принято решение всё равно выводить результаты, чтобы пользователю не было грустно, что он вообще ничего не нашёл и он мог по крайней мере почитать другие классные порошки, раз уж не нашлось такого, как он искал.

В результате BM-25 оказался хорошим инструментом для дословного поиска без всяких додумываний. 

Word2vec всё ещё неплохо учитываем дословное сходство, но дополнительно к этому может найти именно семантически близкие к запросу документы. Например, найти на запрос "кот" документ со словом "кошка" или на запрос со словом "запрет" такой документ:
<blockquote>

у ягуара вопиюще

неокупаемый навоз

но от греха введемте квоту

на ввоз
</blockquote>

А fasttext, несмотря на возлагавшиеся на него надежды, показал себя плохо. Благодаря добавленному улучшению он по крайней мере находит некоторые документы с полностью совпадающими словами, если такие есть, но при этом в остальном, в отличие от word2vec, то, что fasttext определяет как близкие запрос и документ, в реальности едва ли как-то похоже по смыслу.

Говоря о времени поиска, оно оказалось совсем небольшим у BM-25 (около 0.01 секунды), значительно большим для word2vec (около 2.9 секунд) и очень долго для fasttext (около 30-70 секунд — скорее всего, из-за величины загружаемой fasttext модели).

## CLI (Command Line Interface)

Реализован интерфейс командной строки с использованием библиотеки `argparse`.

1. Положить файл с моделью `cc.ru.300.bin` в ту же папку, что и файл `cli.py`.

2. Открыть командную строку.

3. Указать в командной строке путь к папке с файлом `cli.py`. Например, `cd C:\Users\ttais\Documents`.

4. Установить все требуемые библиотеки командой `python -m pip install numpy pandas nltk pymorphy3 scikit-learn gensim fasttext rank-bm25`, затем `python -m nltk.downloader punkt punkt_tab`.

5. Запустить поиск с нужными параметрами командой `python cli.py --query "<текст запроса>" --index <тип индекса> --top_k <k>`.

Примеры проверявшихся запросов: 
- `python cli.py --query "кот олега" --index bm25 --top_k 7`
- `python cli.py --query "кот олега" --index word2vec --top_k 7`
- `python cli.py --query "кот олега" --index fasttext --top_k 7`
- `python cli.py --query "запрет таисьй" --index bm25 --top_k 7`
- `python cli.py --query "запрет таисьй" --index word2vec --top_k 7`
- `python cli.py --query "запрет таисьй" --index fasttext --top_k 7`

На вход подается:
- текст запроса (`--query`)
- тип индекса (`--index`)
- (опционально, по умолчанию 5) желаемое количество документов в выдаче (`--top_k`)

На выходе:
- время выполнения поиска
- для каждого документа в выдаче:
  - индекс документа в корпусе
  - оценка сходства
  - текст документа
